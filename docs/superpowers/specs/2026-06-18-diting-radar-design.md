# 谛听（Dìtīng）· 个人技术情报雷达

> 地藏菩萨的坐骑，伏地侧耳便知三界、辨真假善恶。每天三次"听遍"全网，把和你脑力劳动相关的新东西叼回来，
> 并且**先砍掉你已经知道的**，只留下对你是新的、值得看的。

---

## 1. 一句话

夜里/盘中读你最近几天的**真实脑力劳动记录** → 生成精准查询 → 全网爬 →
**回查你已有的知识、砍掉你已经知道的** → 合成一份"为什么这条对你重要"的情报 →
按三个时段（10:00 / 14:00 / 18:00）各发一份，**飞书推精选 + Obsidian 存全文**。

## 2. 目标 / 非目标

### 目标
- 主动、个性化：不靠手填关键词，**直接读你每天做了啥**来决定爬什么。
- 高信噪比（precision-first）：宁缺毋滥，没好货就如实说"今天这块没值得看的"，绝不硬凑。
- 沉淀闭环：每天的情报落进 Obsidian 知识库，能被 graphify 关联、被未来检索。
- 用最好的脑子：**全程 DeepSeek V4 Pro**，每一步都顶配。

### 非目标（YAGNI，明确不做）
- ❌ 不做二奢生意情报（比价/选品/竞品）——那是另一台引擎，本项目只管**个人技术/科研**。
- ❌ v1 不做"点有用/没用自学习"的反馈闭环（留 v2）。
- ❌ v1 不做中文生态源（知乎/微信技术帖）——先以西文技术源为主，需要再加。
- ❌ 不碰本地模型推理（不抢 Mac Studio 8088 的内存，那条铁律自动绕开）。

## 3. 核心洞察：为什么这个产品对你特别成立

市面上"个性化每日爬虫"最难的一环是 **"怎么知道你今天对啥感兴趣"** ——
一般要么让你手填关键词，要么翻浏览器历史瞎猜。

而你这套基础设施每天都在**自动、结构化地**记录你的脑力劳动，所以"输入信号"是**现成且高质量**的：

| 信号源 | 内容 | 信号密度 |
|---|---|---|
| claude-mem 观察日志 | 带标题的工作流水（"ytst M3 微调上生产"） | 高（标题级，便宜） |
| Qdrant V3 向量记忆 | 近几天的 decision / solution / debug / 悬而未决 | 高 |
| Obsidian 会话记录 | 完整来龙去脉：纠结什么、为何这么决定、卡在哪 | **最高（最肥）** |
| Graphiti 实体时间线 | 你在用哪些模型/工具/品牌、何时开始用 | 中（喂趋势保鲜） |

这就把整条流水线最贵的一步（"理解用户兴趣"）变成了**几乎免费且精准**。

## 4. 总体架构（6 段流水线）

```
①信号层 → ②查询生成 → ③爬取 → ④去重·新颖度 → ⑤合成·排序 → ⑥投递
  读你做了啥   翻成查询    全网抓    砍掉已知的     标"为何重要"   飞书+Obsidian
```

三个"镜头"（科研雷达 / 悬而未决+反方 / 趋势保鲜）**共用 ①③④** 这套引擎，
差异只在 **② 查询角度** 和 **⑤ 合成 prompt**。所以"三镜头"几乎不增加成本——
难的是引擎本身，镜头只是换个提问角度。

## 5. 三份报告（时段 × 镜头 × 信号窗口 × 通道）

| 时间 | 镜头 | 读什么信号（窗口） | 爬来回答什么 | 通道 |
|---|---|---|---|---|
| **10:00** | 🔭 科研雷达 | 截至昨天的活 + 长期关注清单 | "有没有更好的方法 / 我是不是在重复造轮子 / 这方向最新论文" | 飞书+Obsidian |
| **14:00** | 🧩 悬而未决 + 反方 | 卷入**今早**的活 + 历史 TODO/阻塞 | "你卡的那点别人怎么解的" + **唱反调**（X 的缺点/反例/为什么不该用 X） | 飞书+Obsidian |
| **18:00** | 🛰️ 趋势保鲜（版本哨兵） | 你整套技术栈/工具/skill 的实体清单 | "你在用的 X 出了新版 vY / 某项目大更新 / 某法已被弃用" | 飞书+Obsidian |

**越晚的报告越新鲜**：18:00 能反映你今天一整天的进展，不只是"切三块"，而是滚动卷入当天活动。

每份独立成稿、各自能挖深（一份塞三镜头必然每个浅，拆三份才"够详细"）。

## 6. ① 信号层详解（"肥料"从哪来）

兴趣 = **动态肥料**（你最近做了啥）+ **静态肥料**（你长期依赖的栈/工具）+ **长期关注清单**（累积画像）。

### 6.1 动态肥料（每天变）
- claude-mem 观察日志（近 3–7 天，带标题的工作流水）
- Qdrant V3 记忆（近 3–7 天的 decision / solution / debug / feedback / 悬而未决）
- **Obsidian 会话记录**（近几天，最肥）—— 用 DeepSeek V4 Pro 蒸馏出：话题 / 实体 / 未决问题 / 当天决策

### 6.2 静态肥料（你长期依赖的栈，慢变）
- **常用工具 & skill**：graphify、OpenClaw、gbrain、lark-cli 全家桶、你高频用的 skill → 盯它们上游更新
- **本地模型底座 GitHub 项目**：MLX 全家桶（mlx / mlx-lm / mlx-vlm）、vLLM / vllm-mlx、**llama.cpp**、omlx…
- **你在跑的模型**：DeepSeek、Qwen、Gemma…
- **救命手册（cpsk）+ 救命手册 pro（setup-kit-pro）** 里的资料 → 从中挖出你依赖的**整套技术栈**当种子

### 6.3 长期关注清单（profile，累积画像）
- 一份可读可改的清单（如 `state/interest_profile.yaml`），种子来自 6.2 的静态肥料 + 高频实体。
- 每天自动从动态肥料里把反复出现的实体"喂胖"这份清单；也允许你手动增删。
- 作用：**今天没碰某方向也帮你盯着**（解决"日记式只看今天会空报"的问题）。

### 6.4 信号层产物
一个结构化的 **"今日兴趣画像"** JSON：
`{ 热话题[], 用到的工具/实体[], 悬而未决的问题[], 当天的决策[], 长期关注清单[] }`

## 7. ② 查询生成层（三镜头各自的提问角度）

DeepSeek V4 Pro 把"兴趣画像"翻成针对性查询，每镜头不同角度：

- **🔭 科研雷达**：`<话题> 最新论文 / SOTA / 更好的方法 / benchmark 2026`，目标源 arXiv / GitHub / HN / 英文 blog。
- **🧩 悬而未决 + 反方**：
  - 正向：把每条 TODO/阻塞 翻成 `how to <问题> / <问题> 别人怎么解 / best practice`
  - 反向（唱反调）：`<你今天的决策/选型> 的缺点 / 反例 / 为什么不该用 / 失败案例`
- **🛰️ 趋势保鲜**：拿关注清单 + Graphiti 实体名 → `<repo> releases / changelog / v? / deprecated`，**直接打 GitHub Releases / Tags API 拿版本号**。

## 8. ③ 爬取层（尽量复用现成的）

| 源 | 用途 | 接入方式 |
|---|---|---|
| arXiv API | 学术新论文 | 官方 API（query + 时间过滤） |
| GitHub API | trending / releases / tags / 大 issue | REST（releases、search/repositories sort=updated） |
| Hacker News | 技术圈讨论 | Algolia HN Search API |
| searxng | 通用元搜索 | 你已接的 searxng MCP / 本地实例 |
| brave / exa | 兜底网搜 / 语义搜 | 现成 MCP |
| scrapling / defuddle | 抓正文、去噪 | 现成工具 |
| deep-research skill | 需要深挖时的多源核查 | 可选，按需触发 |

爬取层只负责"把候选拉回来"，**判断价值放到 ④⑤**。

## 9. ④ 去重·新颖度层（关键，防止告诉你已知的）

这是 precision-first 的核心。每条候选过两道筛：

1. **跨天去重**：和"已推送库"（见 §13）比对，推过的不再推。
2. **新颖度判定**：把候选摘要丢给 DeepSeek V4 Pro，让它**回查你已有的知识**——
   gbrain（RAG 知识库）+ graphify（vault 文档图谱）+ Qdrant（记忆）——
   判断"**这个我是不是已经知道了**"。已知的降权或砍掉，只留对你是**新**的。

> 这正是《21 个疯狂 OpenClaw 用例》里"dedup against knowledge base"那一招，但用你三套现成记忆系统来做基准。

## 10. ⑤ 合成·排序层

- 用 DeepSeek V4 Pro 合成，每条**必须标注"为什么这条对你重要"**——
  关联到你哪天做的哪件事 / 哪个 TODO / 关注清单里的哪项。
- 排序后：**飞书取精选 top N**（防遗漏），**Obsidian 收全量**（存档）。
- **空报告诚实**：若该镜头当天没有够格的新东西，就明确写"今天这块没值得看的"，不硬凑。
- 每条结构：`标题 / 一句话摘要 / 为什么对你重要 / 来源链接 / 镜头标签`。

## 11. ⑥ 投递层

### 飞书（精选，防遗漏）
- 通过 **lark-cli（im）/ Mac Studio OpenClaw 机器人** 推一条消息到你的飞书。
- 内容：镜头标题 + top N 条（标题 + 一句话 + 为什么重要 + 链接）。简洁，手机一眼扫完。

### Obsidian（全文，存档沉淀）
- 落到 vault 的 `Inbox/`，文件名带日期，例：`2026-06-18 谛听情报.md`。
- **一天一篇**，三个时段各**追加一段**（10:00 科研 / 14:00 悬而未决+反方 / 18:00 趋势保鲜），保持 Inbox 整洁。
- frontmatter 齐全（title / type=note / status / created / last_updated），可被 graphify 关联、被检索。

## 12. 大脑与运行环境

- **大脑：全程 DeepSeek V4 Pro 云端**（你 OpenClaw 主力 `deepseek-v4-pro`，256K 上下文）。
  蒸信号、生成查询、判断新颖度、合成——每一步都用它，不掺本地模型。
- **跑在 Mac Studio**（24 小时常驻），用 **launchd LaunchAgent** 注册三个定时任务。
  - 建议各时段**提前一点开爬**（如 09:30 / 13:30 / 17:30 起爬，整点出报告）。
- **不碰 8088**：本项目只做调度 + 调云端 API，几乎不吃本地内存，自动满足"8088 永不停"。
- **成本**：DeepSeek 云端单价极低，一天 3 份、每条候选都判新颖度也花不了几个钱。

## 13. 数据与状态（持久化）

放在 `~/diting-radar/state/`：
- `interest_profile.yaml` — 长期关注清单（栈/工具/skill/实体），可手动改。
- `pushed.db`（SQLite 或 JSON）— 已推送条目指纹，用于跨天去重。
- `versions.json` — 趋势保鲜的"上次见到的版本号"快照，用于 diff 出"出新版了"。
- `runs/` — 每次运行的原始候选与最终稿留痕，便于排查"为什么推了/没推这条"。

## 14. 错误处理与降级

- 任一信号源/爬源挂了 → **跳过该源、继续跑**，并在报告末尾小字注明"X 源今天没取到"。
- DeepSeek API 失败 → 重试 N 次，仍失败则**当次报告延后/跳过**并飞书发一条简短告警，不静默吞掉。
- 爬取全空 / 全是已知 → 正常出"今天这块没值得看的"，**不算错误**。
- 所有外部输入（API 返回、网页正文）视为不可信，做基本校验再喂给 LLM。

## 15. v1 范围 vs v2

**v1（先跑通核心）**
- 三镜头 + 三时段 + 双通道 + 跨天去重 + 新颖度回查 全做。
- 关注清单先用"静态肥料种子 + 自动喂胖"，手动可改。

**v2（之后再说）**
- "点有用/没用"反馈闭环 → 自动调关注清单和过滤阈值（视频里那个 self-improving filter）。
- 中文生态源（知乎/公众号技术帖）。
- 二奢生意情报镜头（另一台引擎，若要并入）。

## 16. 成功标准 / 验收

- 三个时段能各自**自动产出并送达**飞书 + Obsidian。
- 报告里每条都**关联得回你做过的事**（不是泛泛的科技新闻）。
- **新颖度真有效**：抽查报告条目，确认不是你早就知道的东西。
- 连续一周自测：你愿意每天点开看，而不是当垃圾忽略（precision 达标）。

## 17. 已定决策（2026-06-18 拍板，可改）

1. **项目名** ✅ **谛听（Dìtīng）**，目录 `~/diting-radar`，飞书标题用"谛听"。
2. **飞书发到哪** → 默认：**私聊机器人**（发你本人，Mac Studio OpenClaw 的 ownerAllowFrom 已是你），不打扰别人。
3. **DeepSeek V4 Pro 走哪个 key** → 默认：**复用 OpenClaw 那套** `deepseek-v4-pro`（同 key/endpoint），不再单配。
4. **关注清单初始种子** → 默认：**自动扫** `CLAUDE.md` + cpsk / setup-kit-pro 文档 + `Mac Studio AI 底座/` 手册 + Graphiti 高频实体，生成初版，你随时可改。

> 以上 2–4 是"你看着定"采纳的默认值，实现计划评审时可逐条推翻。

## 18. 里程碑拆分（给后续 writing-plans 用）

1. **信号层**：跑通"读 claude-mem + Qdrant + Obsidian 会话记录 → 蒸出今日兴趣画像"。
2. **关注清单**：从静态肥料种子 + 自动喂胖，建 `interest_profile.yaml`。
3. **爬取层**：接 arXiv / GitHub / HN / searxng，拉回候选。
4. **去重·新颖度**：跨天去重 + gbrain/graphify/Qdrant 回查。
5. **合成·排序**：DeepSeek V4 Pro 出"为什么对你重要"的稿，分镜头。
6. **投递**：飞书 + Obsidian 双通道。
7. **调度**：launchd 三时段定时，端到端串起来。
8. **自测一周**，按 precision 调阈值。
